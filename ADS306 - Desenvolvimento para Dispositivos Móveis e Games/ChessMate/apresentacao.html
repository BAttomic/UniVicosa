Camada de domínio · 1/4

O tabuleiro como estado imutável

Lances legais do cavalo em e4, gerados pela engine

Cada lance devolve uma nova matriz 8×8 — nunca muta a anterior. Isso deixa a IA explorar milhares de posições hipotéticas sem corromper o tabuleiro real, e torna o engine trivial de testar.

⚙️ chessEngine

  • Geração de lances pseudo-legais por peça
  • Filtragem de lances que deixam o rei em xeque
  • Lances especiais: roque, en passant, promoção
  • executeMove() retorna um tabuleiro novo (puro)

Camada de domínio · 2/4

Legalidade: xeque, mate e afogamento

Um lance só é legal se, depois de aplicado, o próprio rei não fica atacado. A ausência de lances legais decide o fim de jogo: em xeque é mate; sem xeque é afogamento (empate). O engine também cobre repetição tripla e regra dos 50 lances.

features/chess/logic/chessAI.js

moves = getAllMovesForColor(board, color, { lastMove, castlingRights });

if (moves.length === 0) {              // nenhum lance legal disponível
  if (isInCheck(board, color)) return -MATE_SCORE - depth;  // xeque-mate
  return 0;                        // afogamento → posição neutra (empate)
}

Descontar depth do mate faz a IA preferir o mate mais rápido.

Inteligência artificial · 3/4

A busca: negamax com poda alfa-beta

A IA olha 3 meios-lances à frente (plies) numa árvore de possibilidades. Negamax é a variante simétrica do minimax — um único código serve para os dois lados, pois pontuação(eu) = −pontuação(adversário). A poda alfa-beta abandona ramos que já não podem melhorar o resultado, cortando a árvore drasticamente.

features/chess/logic/chessAI.js — negamax()

function negamax(board, depth, alpha, beta, color, last, castling) {
  if (depth === 0) return evaluatePosition(board, color, castling);

  let best = -Infinity;
  for (const { fromRow, fromCol, move } of orderMoves(board, moves)) {
    const next  = executeMove(board, fromRow, fromCol, move);
    const score = -negamax(next, depth - 1, -beta, -alpha, opponent, ...);
    if (score > best)  best  = score;
    if (best  > alpha) alpha = best;
    if (alpha >= beta) break;        // ✂ corte alfa-beta: ramo inútil
  }
  return best;
}

Inteligência artificial · 4/4

Como a IA julga uma posição

Nas folhas da árvore, evaluatePosition dá uma nota à posição somando material (peão=100 … dama=900) com tabelas posicionais — bônus por colocar cada peça numa boa casa (cavalo no centro, rei protegido na abertura, rei ativo no final).

chessAI.js — evaluatePosition()

const v = MATERIAL_VALUE[p.type]
        + pieceSquareBonus(p, row, col, endgame);
score += p.color === WHITE ? v : -v;
// perspectiva do lado a mover (negamax):
return color === WHITE ? score : -score;

Detalhe de engenharia

Por que a IA joga rápido (e varia)

⚡ Ordenação MVV-LVA

A poda alfa-beta corta mais quando os melhores lances vêm primeiro. Ordenamos capturas por vítima mais valiosa, agressor mais barato: 

priority = 10 * MATERIAL_VALUE[victim.type]
         -      MATERIAL_VALUE[attacker.type];

🎲 Desempate aleatório

Entre lances de nota equivalente, a IA sorteia um. Sem isso ela repetiria sempre a mesma abertura — o acaso dá variedade às partidas. 

best = candidates[
  Math.floor(Math.random() * candidates.length)
];

Material + tabelas + poda + ordenação = uma IA competente rodando em milissegundos no celular.
