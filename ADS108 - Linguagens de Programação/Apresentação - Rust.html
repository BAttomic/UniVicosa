1. Histórico

Origem e Criação

  • Criador: Graydon Hoare, como projeto pessoal (2006).
  • Adoção: A Mozilla patrocinou o projeto em 2009.
  • Lançamento Oficial: Versão 1.0 lançada em 2015.

Objetivo Inicial

Resolver o problema crônico de falhas de segurança de memória no navegador Firefox, criando uma linguagem para substituir o C++ sem perder performance.

Árvore Genealógica

Influenciada por:

  • C++ (sintaxe básica e performance)
  • ML / Haskell (sistema de tipos forte, pattern matching)
  • Erlang (conceitos de concorrência)

Influenciou:

  • Swift (Apple) - adotou conceitos de segurança.
  • WebAssembly - Rust moldou o padrão Wasm.

2. Ferramentas: Cargo e Crates.io

Cargo: O Maestro

Diferente de linguagens antigas que exigem Makefiles complexos, o Rust vem com o Cargo.

  • cargo new: Cria o esqueleto do projeto.
  • cargo build: Baixa dependências e compila.
  • cargo test: Roda testes automatizados nativamente.

Crates.io (O "NPM" do Rust)

Bibliotecas em Rust são chamadas de Crates (caixotes).

Existe um repositório centralizado oficialmente suportado extremamente rico, garantindo que você não precise reinventar a roda. Tudo é configurado em um arquivo simples chamado Cargo.toml.

3. Características da Linguagem

Paradigmas e Compilação

  • Compilada: Compila diretamente para código de máquina via LLVM. Não tem máquina virtual (JVM).
  • Multiparadigma: Imperativa, Funcional e Orientada a Dados.
  • Tipagem: Forte e Estática (com inferência inteligente).

Sistema de Tipos

  • Tipos Primitivos: Inteiros (i32, u64), Float (f32), Booleanos, Chars.
  • Tipos Compostos: Tuplas, Arrays (tamanho fixo).
  • Novos Tipos: Feita através de structs e enums.

Curiosidade: O Mindset de Memória

O dev de Rust pensa na memória em tempo de compilação através do Ownership e Borrowing.

A Luta com o Compilador

Dois "donos" não podem ter a mesma variável na memória heap.

let s1 = String::from("Olá");
let s2 = s1; // A posse MUDOU para s2

// println!("{}", s1); // ERRO! 
// s1 não é mais válida.

Como Reestruturar o Pensamento?

  • O dev se pergunta: "Quem é o dono deste dado?"
  • Se outra função precisa ler, nós "emprestamos" (Borrowing) com referências (&).
  • Regra de Ouro: Infinitas referências de leitura, OU apenas uma mutável. Fim das "corridas de dados"!

Curiosidade: O Lado "Obscuro" (Unsafe)

Se o compilador bloqueia tudo que é perigoso, como interagir com hardware (acesso direto à memória)?

O bloco unsafe {}

O Rust permite "desligar" algumas proteções dentro de um bloco específico:

  • Ler/escrever em ponteiros crus (raw pointers).
  • Chamar bibliotecas legadas (C/C++).
  • Acessar hardware de baixo nível diretamente.

Isolando o Perigo

A vantagem disso não é "nunca ter bugs", mas sim isolar onde eles ocorrem. Se houver uma falha crítica de memória, ela só pode estar dentro de um bloco unsafe, reduzindo a busca de milhões de linhas para algumas dezenas.

5. WebAssembly (Wasm): O Futuro do Frontend

O Rust não ficou restrito ao Backend. Ele está mudando a forma como usamos o navegador.

O que é WebAssembly?

Wasm é um formato binário que roda nativamente dentro do browser (Chrome, Edge, Firefox) ao lado do JavaScript, mas com performance de linguagem compilada.

  • Rust é hoje a linguagem número 1 para gerar código Wasm, graças à sua ausência de Garbage Collector.

Aplicações Reais (Frontend Pesado)

Empresas como Figma e Adobe compilam programas inteiros (edição gráfica, renderização 3D, edição de vídeo) em Rust/Wasm para que rodem lisos direto na aba do seu navegador, sem precisar instalar nada no Windows.

6. Código: Tipos (Struct e Enum) + O Segredo do "!"

Sem classes tradicionais de POO, o Rust une dados com structs e enums.

enum Status {
    Ativo,
    Suspenso(String),
}

struct Usuario {
    nome: String,
    status: Status,
}

impl Usuario {
    fn novo(n: String) -> Usuario {
        Usuario { nome: n, status: Status::Ativo }
    }
}

O Segredo da exclamação "!"

No comando println!("Olá {}", nome); o "!" indica que isso não é uma função, mas uma Macro.

Macros expandem o código antes de compilar. Isso permite que o compilador verifique se você passou a quantidade correta de variáveis para imprimir, prevenindo crashes que ocorreriam no printf do C.

6. Código: Tratamento de Exceções

Rust não possui blocos try/catch. Exceções são retornadas como valores através do enum Result<T, E>, forçando o dev a tratar o erro.

use std::fs::File;

fn abrir_arquivo() {
    // File::open retorna Ok(arquivo) ou Err(erro)
    let resultado = File::open("dados.txt");

    // 'match' obriga a tratar todos os cenários possíveis
    match resultado {
        Ok(_arquivo) => println!("Arquivo aberto com sucesso!"),
        Err(erro) => println!("Falha ao abrir: {}", erro),
    }
}

6. Código: Concorrência (Threads)

O lema é "Fearless Concurrency". O compilador impede o acesso simultâneo indevido aos dados.

use std::thread;
use std::time::Duration;

fn main() {
    let handle = thread::spawn(|| {
        for i in 1..4 {
            println!("Na thread paralela: {}", i);
            thread::sleep(Duration::from_millis(500));
        }
    });

    println!("Na thread principal!");
    handle.join().unwrap(); // Espera paralela terminar
}

6. Código: Polimorfismo (Traits)

Para compartilhar comportamentos (POO/Polimorfismo), utilizamos Traits (semelhante às Interfaces no Java).

// A Interface de comportamento
trait Veiculo {
    fn acelerar(&self);
}

struct Carro;
struct Moto;

// Implementando o Trait para Tipos Diferentes
impl Veiculo for Carro {
    fn acelerar(&self) { println!("Carro: Vrum Vrum!"); }
}

impl Veiculo for Moto {
    fn acelerar(&self) { println!("Moto: Ran dan dan!"); }
}

8. Estudo de Caso e Sustentabilidade

Case Discord (Adeus, GC)

O Discord migrou um serviço vital (Read States) de Go para Rust. Em Go, o Garbage Collector causava enormes picos de latência (travamentos). Em Rust, com o Ownership liberando memória na hora, o uso de CPU despencou e a latência ficou perfeitamente reta.

Green Computing (ESG)

Sustentabilidade em TI é real. Um estudo recente da AWS (Amazon) demonstrou que migrar servidores de linguagens interpretadas (como Python ou Java) para Rust reduziu o consumo de energia dos Data Centers em até 50%.

O Rust não economiza só tempo e dinheiro; ele gasta literalmente menos eletricidade.

9. O Apelo da Casa Branca (EUA)

"A escolha da linguagem virou uma questão de Segurança Nacional."

O Relatório ONCD (Fev/2024)

O governo dos Estados Unidos emitiu um relatório oficial clamando para que as Big Techs abandonem projetos em C e C++ devido às intermináveis falhas de injeção de memória (que viram portas para hackers).

O relatório recomendou enfaticamente a transição para linguagens **Memory-Safe**, colocando o Rust como o principal candidato para o futuro seguro da tecnologia mundial.
